Laboratory Work

Mobile Dev Labs

Lab 1

User Interface Design

Built foundational UI screens for a mobile application using Jetpack Compose, focusing on layout structure, colour contrast, and typography that meets accessibility guidelines.

Crafting the first UI screens taught me how even small spacing decisions affect the entire feel of an app. I learnt to think from the user's eyes first, and realised that clean design and clean energy share the same principle — efficiency with purpose.
Lab 2

Interaction & Navigation Flow

Explored user interaction patterns including gesture navigation, button states, and animated transitions using Compose's state management system.

This lab shifted how I think about user flow — every tap is a decision the user makes, and my job is to make that decision feel effortless. Smooth interactions build trust, just like a reliable clean energy grid builds confidence in renewable solutions.
Lab 3

Material Design System

Applied Material Design 3 principles — adaptive colour schemes, dynamic theming, and component libraries — to create a polished, production-ready look and feel.

Material Design gave me a shared language with other developers. I discovered that great design systems are like power grids — standardised, scalable, and only truly powerful when everything is connected and consistent.
Lab 5

Room Database Integration

Integrated the Room persistence library to store and retrieve app data locally on the device, using Entity, DAO, and Database components alongside Kotlin Coroutines for safe background operations.

Working with Room made me appreciate how important structured local storage is for a reliable user experience. Seeing data persist after closing the app was a turning point — it made the app feel truly production-ready, much like how a reliable energy grid keeps running even when demand fluctuates.
Lab 4

Navigation Architecture

Implemented multi-screen navigation using the Navigation component in Jetpack Compose, including bottom navigation bars, deep links, and back-stack management.

Building a navigation system forced me to think architecturally rather than screen-by-screen. A well-wired app, like a well-wired building, lets energy flow to exactly where it's needed without waste.

Capstone Projects

VoltPath

⚡ VoltPath — Working Features

Project 1  ·  Lab 4 + Core App Features

View on GitHub
Problem Statement: Many Malaysians lack an easy way to locate nearby EV charging stations, compare tariffs, and plan routes that minimise energy consumption — creating range anxiety and slowing EV adoption.

Lessons Learned

Building the core features of VoltPath taught me how to balance real-time data with offline resilience, ensuring the app remains useful even in low-connectivity areas. I also deepened my understanding of Room database design and the importance of separating data layers cleanly from the UI — a principle that paid dividends when debugging race conditions.

🔋 VoltPath — API + Sensors + Cloud

Project 2  ·  Lab 5 + Retrofit + Firebase + ML Kit + CameraX

View on GitHub
Problem Statement: EV drivers need live station availability, smart energy-efficiency routing, and camera-assisted charger identification — features requiring seamless integration of cloud APIs, device sensors, and machine learning.

Lessons Learned

Integrating Retrofit, Firebase Firestore, and ML Kit in a single project revealed how critical a well-defined repository pattern is when data arrives from multiple asynchronous sources. I learnt to use Kotlin Coroutines and Flow to keep the UI reactive without blocking the main thread, and discovered that CameraX dramatically simplifies real-world ML deployment on Android. This phase brought VoltPath closest to a production-grade app.

Learning Journey

Reflection

When I started this course, "mobile development" felt like a distant, complex world — a stack of unfamiliar terms like Composables, Coroutines, and ViewModel. By Lab 1, I had built my first working screen; by Project 2, I had shipped an app that reads live data from the cloud, uses the device camera for ML inference, and routes users to the nearest clean-energy charging station.

The most transformative lesson was not technical — it was the discipline of thinking about real users. Every feature I built for VoltPath had to answer one question: does this make sustainable transport more accessible? That SDG 7 lens kept me grounded whenever I was tempted to add complexity for its own sake.

I leave this semester not just with working code, but with a genuine belief that software engineers are infrastructure builders — we lay the digital wires that future solutions run on. I intend to keep building on that foundation, one commit at a time.
